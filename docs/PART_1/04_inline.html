4  Inplace Operations

Daisy ficou emocionada com seu progresso na criação da sua agenda semanal, e decidiu fazer um programa para o mês inteiro. Porém, dessa vez, ela quer fazer algo diferente. Ela quer usar uma variável como Input, aonde serão armazenados os resultados.

Ela decidiu fazer isso, pois ela descobriu que em Julia, qualquer funções pode alterar o valor de vetores ou matrzes, mas para evitar confusão, é costume nomear funções que modificam suas entradas com um ! no final do nome, assim, Daisy critou a função calendario_mensal!.

Daisy pensou em aproveitar o código para calcular a agenda semanal, afinal, é aplicar a mesma função para todas as semanas do respectivo mês. Martin, seu mestre, disse que isso funcionaria, mas não é uma boa idéia. Já que cada mẽs tem quantide de semanas diferentes; seria uma ideia melhor (vai ter menos bugs) se Daisy criasse uma solução simples para Fevereiro, pois possui (em geral) 28 dias ( 28 dias = 4 semanas de 7 dias).

Daisy estava fazendo uma funçãoq eu só funcionava em um caso especifico, para garantir que o usuário não usasse outro mês por engano, ela decidiu que sua função iria gerar um erro, e não iria funcionar, se a matriz com as respostas (agenda_fevereiro) não tivesse as dimensões corretas. Para isso, ela usou uma macro chamada @assert. Macros são funções que podem criar códigos, existem muitas delas, mas nesse exemplo, Daisy só precisava saber que se as dimensões da matrizes estivessem erradas, a função não iria funcionar.

Sintaxe: @assert

@assert só se manifesta como um erro se a condição for false, do contrário, ela não faz nada.

function teste_valor(x)
    @assert x > 5
    return "OK"
end
@show teste_valor(10)
begin
    try
        teste_valor(1)
    catch
        println("Função falhou, e precisou de tratamento.")
    end
end;
teste_valor(10) = "OK"
Função falhou, e precisou de tratamento.

Mais uma dica do seu mestre, era para não armazenar sim ou não nas matrizes, já que isso ocupa muita memória, e em aplicativos mobile, Daisy deveria se preocupar em ter programas muito leves. A solução é armazenar apenas 0 ou 1. Contudo, essa solução não é perfeita, e se Daisy precisasse no futuro armazenar 0.0 e 1.0? Felizmente em Julia é possível resolver isso criando variáveis do tipo certo, no lugar de criar o valor 1, Daisy usou a função one(), que criar um 1 do tipo da variável que ele recebe na entrada, esse 1 correto, Daisy chamou de sim.

Sintaxe: one()

one recebe um tipo, se você não sabe o tipo de uma variável, pode descobri-lo com eltype.

@show one(Int64)
@show one(Float64)
@show one(Bool)
@show one(eltype(1.0))

@show one(eltype(1.0)) == one(Float64);
one(Int64) = 1
one(Float64) = 1.0
one(Bool) = true
one(eltype(1.0)) = 1.0
one(eltype(1.0)) == one(Float64) = true

Daisy então escreveu

function agenda_mensal!(agenda_fevereiro)
    qtd_semanas, qtd_dias = size(agenda_fevereiro)

    # verifica que a matriz de entrada é de fevereiro
    # se alguma confição for falsa, trava o código
    @assert qtd_dias == 7
    @assert qtd_semanas == 4

    # o 'sim' pode ser de qualquer tipo
    sim = one(eltype(agenda_fevereiro))

    for i = 1:qtd_semanas
        for j = 1:qtd_dias
            permissão = rand() < 0.5
            if permissão == true
                agenda_fevereiro[i,j] = sim
            end
        end
    end
    return nothing
end
agenda_fevereiro = zeros(Bool, 4, 7)
agenda_mensal!(agenda_fevereiro)

display(agenda_fevereiro)
